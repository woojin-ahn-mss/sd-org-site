SD/Console

Search & Discovery 실의 운영 사이트(8 페이지)를 받치는 디자인 시스템입니다. 카드 대신 룰(border-bottom)로 위계를 짜고, 데이터는 모노 폰트로 묵직하게 다룹니다. 컬러는 골드 1개와 위험 빨강만 사용해 정보 가독성에 집중합니다.

1. 색

Dark가 1차 테마, Light는 동등하게 지원합니다. 컬러는 의도적으로 적게: 골드(액센트) 하나와 위험 빨강이 거의 전부입니다.

25. 사용 원칙

이 시스템을 유지하기 위한 6가지 룰.

RULE 01

카드 대신 룰

흰 배경 박스를 쌓지 않습니다. 정보 위계는 위·아래 1px --rule 라인과 mono uppercase 섹션 헤드로만 잡습니다. 박스가 필요해 보이면 마진을 더 주거나 룰을 바꿉니다.

RULE 02

데이터는 mono

티켓 키, 날짜, 숫자, 카운트, 분수, 진척률 — 모두 JetBrains Mono. 한글 본문/타이틀은 Pretendard. 두 가족만 사용합니다.

RULE 03

컬러는 골드 하나

액센트는 --accent (#e3b340) 하나입니다. "오늘", 활성 탭, 키 링크, 진행 게이지에만 사용. 다른 곳에 컬러를 칠하지 않습니다. 위험은 빨강, 완료는 그린만 사용.

RULE 04

모서리는 거의 0

border-radius는 0~2px. 박스, 칩, 버튼 모두 sharp. "친근함" 보다 "운영 도구"의 인상을 우선합니다.

RULE 05

페이지 헤더는 editorial

키커 → 32px 타이틀 → 14.5px 리드 문장. 리드는 항상 "이 페이지를 열었을 때 즉시 알아야 할 핵심 문장"으로 작성합니다. 숫자는 mono 골드로 강조.

RULE 06

밀도 우선

운영 도구는 매일 보는 화면입니다. 여백을 늘려 "고급스러워" 보이게 만들지 말고, 한 화면에 필요한 정보를 정확히 넣습니다. 행 padding 11~12px, KPI 카드 X.
